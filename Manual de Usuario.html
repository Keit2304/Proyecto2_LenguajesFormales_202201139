Descripción del Proyecto

El proyecto consiste en la elaboración de una herramienta que permita el diseño y creación de sentencias de bases de datos no relacionales de una forma sencilla. La aplicación tendrá un área de edición de código y un área de visualización de la sentencia final generada.
Cuando ya se cuente con las sentencias creadas inicialmente, se procederá a realizar la compilación respectiva lo que generar las sentencias de MongoDB que serán mostradas en el espacio de resultados que posteriormente se podrán aplicar a un entorno adecuado a MongoDb.

Objetivos

  • Objetivo General
    • Combinar los conocimientos adquiridos en el curso y en los otros cursos de sistemas, para crear un compilador que traduzca el lenguaje especificado y lo transforme en Sentencias de Bases de Datos No Relacionales.
  • Objetivos Específicos
    • Crear una herramienta que permita el diseño de sentencias de base de datos no relacionales de una forma sencilla para el usuario.
    • Diseñar y construir un compilador que permita compilar archivos de entrada y visualizar el resultado en un entorno externo.
    • Desarrollar la habilidad del estudiante para elaborar proyectos en base a una adecuada planificación para que aprendan la manera en la que tienen que trabajar.

Requisitos del Sistema

  • Python 3.x instalado en su sistema.
  • Conocimientos básicos de programación en Python.
  • Familiaridad con el formato de código NoSQL y MongoDB.

Instalación

No se requiere una instalación especial para utilizar esta aplicación. Solo asegúrese de tener Python 3.x instalado en su sistema.

Funcionalidades

La aplicación proporciona las siguientes funcionalidades:

  • Nuevo: Permite crear un nuevo archivo.
  • Abrir: Permite abrir un archivo existente.
  • Guardar: Permite guardar el archivo actual.
  • Guardar Como: Permite guardar el archivo actual con un nombre diferente.
  • Salir: Permite cerrar la aplicación.
  • Generar sentencias MongoDB: Realiza el análisis léxico y sintáctico del código NoSQL en el área de edición de código y traduce las sentencias válidas a sentencias MongoDB.
  • Ver Tokens: Muestra un reporte de los tokens encontrados en el código actual.
  • Ver Errores: Muestra un reporte de los errores encontrados en el código actual.

Instrucciones de Uso

  1. Al abrir la aplicación, se presentará una ventana con un área de edición de código y un menú en la parte superior.
  2. Utilice las opciones del menú Archivo para crear, abrir, guardar o guardar como un archivo.
  3. Escriba o pegue su código NoSQL en el área de edición de código.
  4. Utilice la opción “Generar sentencias MongoDB” del menú Análisis para traducir el código NoSQL a sentencias MongoDB.

    5.Utilice las opciones del menú Tokens y Errores para ver un reporte de los tokens y errores encontrados en el código, respectivamente.
